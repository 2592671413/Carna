Detailed Description

template<typename VoxelType, typename TextureUploader, typename BufferType = std::vector< VoxelType >>
class Carna::base::model::GenericVolume< VoxelType, TextureUploader, BufferType >

Generic Volume implementation.

Invariant:
sizeof( VoxelType ) >= 2

Template Arguments:

  • The VoxelType is the data type used to store the value of a single voxel / HU-value.
  • The TextureUploader is a class, which meets the following requirements:
    • It has a public default constructor.
    • It implements a public method with the interface:
      uploadTexture( const Vector3ui& size, const void* bufferPtr ) const
      This method satisfies the requirements documented here.
  • The BufferType is the data type used as voxel container.

template<typename VoxelType , typename TextureUploader , typename BufferType = std::vector< VoxelType >>
virtual void Carna::base::model::GenericVolume< VoxelType, TextureUploader, BufferType >::uploadTexture ( ) const [inline, virtual]

Uploads the data as 3D texture to the video memory.

The implementation must meet the following requirements:

  • The data must be uploaded to the texture object currently bound.
  • The texture must be initialized as a single-channel (red) GL_TEXTURE_3D texture.
  • The HU-values from \([-1024, 3071]\) must be provided as a linearly mapped \([0, 1]\) interval.

Exemplary implementation:

 glTexImage3D( GL_TEXTURE_3D, 0, GL_INTENSITY16
     , size.x, size.y, size.z
     , 0, GL_RED, GL_UNSIGNED_SHORT, buffer_ptr );

Implements Carna::base::model::Volume.
