Detailed Description

Renderer that implements ray marching. This is a technique widely used in 3D graphics for rendering volume data.

Ray Marching

The renderer implementation supplies the volume data as a 3D texture to the OpenGL API. It also computes the front and back face coordinates of each pixel in texture space, as illustrated by the picture below: both coordinate mappings, \( front(x, y) \) and \( back(x, y) \), are implemented as a RGB 2D texture each, whose color channels do carry the corresponding texture space coordinates for any particular texel.

Illustration of the two functions front and back
Note:
Notice that in the above image the two exemplary texture space points are mapped to different screen space coordinates. The algorithm is interested in such points that are mapped to the same screen space coordinates. But illustrating such two points, identified by \(x_1=x_2\) and \(y_1=y_2\), would make them indistinguishable in the image above.

Both coordinate mapping textures are bound to an exclusive texture sampler each by the renderer. The actual ray marching rendering algorithm is implemented diversely, dependent on the specific VolumeRendererMode currently used - regardless of this, it receives the texture samplers to use for sampling the coordinate textures as a configuration object from the renderer.

Usually the implementation will compute a ray in texture space for each screen space pixel \( (x, y) \), that intersects the texture volume: the ray will start in \( front( x, y ) \) and end in \( back( x, y ) \). This ray will be evaluated in equidistant steps, the distance between two successive ray samples being determined by the current sample count setting:

\[ \text{sample distance} = \frac{ \sqrt 3 }{ \text{sample count} - 1 } \]

The default sample count setting is:

\[ \text{sample count} = \sqrt{ (\text{voxels along } x_t)^2 + (\text{voxels along } y_t)^2 + (\text{voxels along } z_t)^2 } \]

See also:
Refer to ''NVIDIA GPU Gems 3'', chapter 30.3, ''Rendering'' for deeper understanding of the technique.

Miscellaneous

When the ray marching is finished by the supplied VolumeRendererMode, a further rendering pass is performed in order to apply a set of optional post processing steps. These include:

  • inverting colors
  • overlaying a black circular mask
  • rotating the image
  • flipping the image horizontally and/or vertically

Usage

There is an example on how to use the volume renderer here.

The VolumeControllerUI class offers an easy-to-use user interface with the purpose of configuring this renderer.

Queries the scene matrix.

This view's scene matrix is defined by:

\[ S = S_0 \cdot M_0 \cdot M_R \]

with

\[ M_R = \begin{pmatrix} 1 & 0 & 0 & r_x - 0.5 \\ 0 & 1 & 0 & r_y - 0.5 \\ 0 & 0 & 1 & r_z - 0.5 \\ 0 & 0 & 0 & 0 \\ \end{pmatrix} \cdot R \cdot \begin{pmatrix} 1 & 0 & 0 & -r_x \\ 0 & 1 & 0 & -r_y \\ 0 & 0 & 1 & -r_z \\ 0 & 0 & 0 & 0 \\ \end{pmatrix} \]

and \(R\) being the rotation offset and \(\vec{r}\) the rotation center in volume units.
