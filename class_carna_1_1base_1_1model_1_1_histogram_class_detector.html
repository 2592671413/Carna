Detailed Description

template<typename HistogramType>
class Carna::base::model::HistogramClassDetector< HistogramType >

Computes classes of a given histogram.

Parameters:
HistogramTypeMust be a Carna::base::model::Histogram class template instance with HistogramType::FieldCodomain as an integral data type.

Since HistogramType::FieldCodomain must be an integral data type, elements of HistogramType::FieldCodomain always can be arranged in some order. From this arises the terminology of lowest or first and highest or last keys, as it is used below.

The usual usage of this class looks like: 

 #include <Carna/base/model/HistogramClassDetector.h>

 template< typename Histogram >
 void printHistogramClasses( const Histogram& histogram )
 {
     typedef Carna::base::model::HistogramClassDetector< Histogram > HistogramClassDetector;
     HistogramClassDetector class_detector( histogram );

     class_detector.compute( HistogramClassDetector::Configuration
         ( -1024 //< first: the lowest key of the histogram
         ,  3071 //< last: the highest key of the histogram
         ,   30u //< gaussian radius
         ,  100u //< minimum width of a typical class
         ,   0.2 //< minimum logarithmic height difference ) );

     for( auto class_itr = class_detector.result().begin(); class_itr != class_detector.result().end(); ++class_itr )
     {
         qDebug() << class_itr->first << " - " << class_itr->last << ( class_itr->typical ? " (typical)" : "" );
     }
 }

For details on the last three arguments of the compute method refer to the algorithm documentation.

Histogram Class Detection Algorithm

The algorithm presented here is based on the assumption that the width of typical histogram classes is lower bound with the threshold \(t \in \mathbb{N}\). Therefore, given the following two conditions are fulfilled, a class \(\left[a;b\right]\) is called typical:

  • The class' width \(b - a + 1\) is larger or equal the threshold \(t\).
  • The logarithmic gap between the lowest and the highest frequency the histogram \( H : K \to \mathbb{N}_0 \) attains within \(\left[a;b\right]\) is greater or equal a certain constant \(h \in \left[ 0; 1 \right]\):

    \[ \exists x \in \left[ a; b \right] : \frac{ \log_{10} H \left( x \right) }{ \max\limits_{x' \in K}\ \log_{10} H \left( x' \right) } \geq h \]

The algorithm partitionates the whole histogram into a gapless sequence of typical and non-typical classes.

Hereinafter all indices are considered to start with 1, not with 0. A logarithmic representation \(L\) of the key counts carried by the histogram counter \(H\) is computed. The lowest and the highest values that are to be recognized by the algorithm must be specified as \(k_\text{min}\) and \(k_\text{max}\):

\[ L \left( k_\text{min}, k_\text{max} \right) = \Big( \log_{10} \big( H \left( k \right) + 1 \big) : k \in \left[ k_\text{min}, k_\text{max} \right] \subseteq K \Big) \]

Second, a smoothed representation \(S\) of \(L\) is computed, utilizing Gaussian kernel \(\mathcal{G}_{\sigma = r/2}\) with radius \(r\). Looking at the derivative of \(S_i\) we identify those \(i\) that do represent local minimums, whereby \(k_\text{min}\) and \(k_\text{max}\) (represented by the first and the last elements of \(S\)) are always considered such local minimums:

\[ M = \Big\{ i : S_{i-1} < 0 \wedge S_i > 0, i \in \big[ 2; \left\| S \right\| \big] \Big\} \cup \Big\{ 1, \left\| S \right\| \Big\} \]

Now the actual class detection begins. Lets consider \(M \subset \mathbb{N}\) to be sorted ascendingly. The algorithm now goes as follows:

  1. Set \(a = M_1\).
  2. For all \(i = 2, 3, \dots, \left\| M \right\| \):
    1. Set \(b = M_i\).
    2. If \( b - a \geq t \):
      1. If \( \big( \max_{j = a}^b S_j \ - \ \min_{j = a}^b S_j \big) / \max S > h \):
        1. If we haven't notated a histogram class yet and \( b > 1 \):
          1. Notate the non-typical histogram class \(\left[ 1, b - 1 \right]\).
        2. Else, if we already have notated at least one histogram class and \(c + 1 < b\):
          1. Notate the non-typical histogram class \(\left[ c + 1, b - 1 \right]\).
        3. Notate the typical histogram class \(\left[ a; b \right]\).
        4. Set \(c = b\).
    3. Set \(a = b + 1\).
  3. If no classes have been notated:
    1. Notate the non-typical histogram class \(\left[ 1, \left\| M \right\| \right]\).
  4. Shift all notated classes by adding \(k_{min}\) as an offset.

It is a side effect of the algorithm that two or more non-typical classes may not occur in a row.
