Detailed Description

Defines the data model.

Data Model

The model's main purpose is to provide access to the volume data: this data consists of a scalar field, which is described by an associated Volume object, and is enriched with some extra informations by the model, that is required for it's visualization, such as the distance between two neighbor voxels. These distances are measured in millimeters and called spacing. Furthermore it is the purpose of the data model to suggest a global windowing, which however does not have to be taken into account by a specific view. A single Scene instance can be used by any number of views.

Besides volume data, the spacing and windowing, the data model can be used to provide additional, custom geometries: these are represented by a set of 3D objects.

It is explicitly not possible to switch the model assigned to a view during the view's lifetime. If this is necessary, the view must be re-instantiated.

Loading Data

There are two ways of creating a Scene instance:

  1. Either you can use one of the pre-defined routines of the SceneFactory, or
  2. you must use the Scene's constructor directly. In this case you will have to define the association with the volume data: particularly you will have to decide whether the data model will take ownership of the volume data (utilizing Composition) or whether the model is only to reference the data (choosing Aggregation). The latter case will allow the volume data to exist independently of the model's lifetime.
Warning:
You must initialize QApplication (or any of it's derivations, such as Carna::Application) before the first usage of Scene. Speaking strictly, you must ensure that QApplication is instantiated by the time that view::SceneProvider::max3DTextureSize is invoked for the first time. Such an invocation is performed by the SceneFactory internally, for instance. If you instantiate Scene by yourself, using its constructor, you might be able to prolong the instantiation of QApplication.

Here are some instantiation examples:

 using namespace Carna;
 using namespace Carna::base;
 using namespace Carna::base::model;

 const float spacingXY = 1.f;
 const float spacingZ  = 1.f;

 // using the constructor (2)

 UInt16Volume volume( Vector3ui( 128, 128, 128 ) );

 Scene model1( new Aggregation< Volume >( volume )
             , spacingXY, spacingXY, spacingZ );

 Scene model2( new Composition< Volume >
         ( new UInt16Volume( Vector3ui( 128, 128, 128 ) ) )
         , spacingXY, spacingXY, spacingZ );

 // using DicomSceneFactory from Carna-DICOM project (1)
 // https://github.com/RWTHmediTEC/Carna-DICOM

 using Carna::dicom::DicomSceneFactory;
 DicomSceneFactory modelFactory;

 std::unique_ptr< Scene > model3( modelFactory.createFromUserInput() );
