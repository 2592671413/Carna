API 2.x Version Log

This page gives you an overview over the recent API changes.

Versioning Policies

The major version number names the overall API version. It is incremented on wide-ranged API changes. There is absolutely no downward compatibility guaranteed: The compatibility of the API is usually broken when this version number is incremented.

The minor version number tells the "local" API version. It is incremeted on such API changes, that do break downward compatibility on the one hand, but do have only restricted, local impact on the other hand. This means that for the library's user there usually are only a few tweaks neccessary - if any - in order to upgrade.

The patch version number is incremeted on such changes that neither of the two prior rules apply to: Full downward compatibility is guaranteed, thus one should always try to use the most recent patch version.

Change Log

API Version 2.0

See API Evolution: From 1.x to 2.x

API Version 2.1

The changes introduced in this release emerged from migration of CRA and DICOM Viewer to the Carna 2.x API.

Added methods:

  • VisualizationEnvironment::paintText
  • VolumeRendererMode::invalidate

API Version 2.1.1

The build system has been switched from QMake to CMake. The method ModelFactory::computeVoidThreshold has been added, in order to to allow an easy estimation of a proper value for Model::setRecommendedVoidThreshold when instantiating Model without ModelFactory.

API Version 2.1.2

The WARN_IF_UNDOCUMENTED flag has been switched to on in Doxyfile, thus yet undocumented entities have been documented. Also several new constraints have been introduced:

  • Pre- and post-conditions on Object3DChooser::selectObject
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::setPositionedObject
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::removePositionedObject
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::setLightAmplitude
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::setLightBias
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::setLighting
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::setLightBackFaceAmount
  • Post-conditions on AbsorptionProjection::AbsorptionProjection::setPickingSaturation
  • Post-conditions on AxialPlane::AxialPlaneControllerUI::setDeletedWhenEmpty

Further tweaks:

  • Span::operator== is now defined through isEqual.
  • Object3DChooser::getSelectedObject3D has been renamed to Object3DChooser::selectedObject3D.

API Version 2.1.3

Fixed bug that caused an access violation within UInt16TextureUploader::uploadTexture in rare cases. Also fixed access violation in Minimal Example that occurred when the opening dialog was canceled.

Added Functionallity

The classes OffscreenDisplay and OffscreenVisualizationEnvironment have been added, introducing easy offscreen rendering support. The environment can operate both, providing an exclusive OpenGL context or sharing the context with any other VisualizationEnvironment, such as that from a regular Display. Thus, exclusive-mode SlicePlane constructor has been marked deprecated.

The Version.h header file has been added, introducing set of new macros:

  • CARNA_MAJOR_VERSION, CARNA_MINOR_VERSION and CARNA_RELEASE_VERSION
  • CARNA_ASSERT_API_VERSION

API Version 2.1.4

This release focuses upon improvements (including bug fixes) to parallelization support and DICOM data loading routines.

API changes:

  • Cleaned up the parallelization API: The Parallelization template has been modified, but also the ParallelizationUI interface has been updated.
  • Dropped the SeriesElement::SeriesElement(const std::string&) constructor.

Bug fixes:

  • Fixed strange assertion failure, that only arised in release mode, when scanning certain DICOM directories.
  • Fixed access violation arising frequently during DICOM directory scanning and switched to dicom-interface-carna version 0.2.0 in this context.
  • Fixed issue with access violation arising after DicomController destruction in rare cases, due to thread lifetime inconsistencies.
  • Fixed the CARNA_FAIL, CARNA_ASSERT and CARNA_ASSERT_EX macros in release mode.

API Version 2.1.5

This release focuses upon bug fixes and testing:

  • Bindings of the FramebufferObject now work across multiple GL contexts as expected.
  • OffscreenDisplay is working now. The bug fixed was making the first frame being rendered - ever by a Renderer instance or after Renderer::setTargetSize - always be 'wrong'. Particularily the frame was renderer with wrong projection matrix, due to the viewport being set up after, but not before, the projection matrix computation. An example on offscreen rendering has been added to the Quick Start Guide page.
  • Added OffscreenDisplayDemo and InteractiveOffscreenDisplayDemo to CarnaInteractiveTest.
  • Fixed bug in Parallelization::run that caused invocations to return too early in rare cases.

API Version 2.1.6

This release fixes the z-spacing recognition within the DICOM directory reading routine. Therefor it now relies on dicom-interface-carna version 0.2.1 instead of 0.2.

API Version 2.2

Although this release focuses on bug fixing and testing, 100% downward compatibility of the API is not given, since several pre-conditions have been added to the Model constructor. However, it is highly unlikely that you are affected by this modification.

The other changes are in detail:

  • An automatical test suite infrastructure has been added to the project, including an unit test of the Model class and a module test for offscreen rendering exemplary. QTestLib has been utilized as the testing framework.
  • Bug has been fixed in OffscreenDisplay (by adding a missing glewInit directive) which could have prevented the class from working as expected on certain systems.
  • Added the BreakForDebug class to control the behavior of the CARNA_BREAK_FOR_DEBUG macro at runtime.
  • The Model::recommendedVoidThresholdChanged signal has been added.
  • Added qRegisterMetaType directive to the Model constructor, making the Model::objectsChanged signal work out of the box.
Warning:
The CARNA_BREAK_FOR_DEBUG macro definition is partially broken in this realease: If used by a client application or library, it is necessary to add either one of the following directives in the same scope, that the CARNA_BREAK_FOR_DEBUG macro is used in. 
 using Carna::BreakForDebug;

 // or

 using namespace Carna;
This issue did not exist prior to API Version 2.2 and has been fixed in API Version 2.2.1.

API Version 2.2.1

Bug fixes:

  • The CARNA_BREAK_FOR_DEBUG macro definition, that was partially broken since API Version 2.2, has been fixed.
  • In prior releases user interface interaction often lead to assertion failures, commonly due to failing post-conditions. This issue has been fixed through an adjustion to the epsilon class, as documented below.

Other modifications:

  • Post-conditions have been added to OffscreenDisplay::resize
  • The double-datatype specialization of epsilon has been changed from \(10^{-8}\) to \(10^{-6}\).

API Version 2.2.2

The vessel enhancement controls (not working properly anyway) have been removed from XRay. Built-in support for factorial and binary volume masks has been added. New methods, types and attributes are in detail:

  • BufferedMask, BufferedMaskAdapter
  • Model::setMask, Model::unsetMask, Model::hasMask, Model::mask, Model::maskExchanged
  • Scene::ResourceContext::maskTextureID, Scene::ResourceContext::updateContext

Currently only the XRay volume rendering mode supports masks. Masks are ignored by renderers that do not support them.

Further added entities:

  • Model::intensityAt
  • Polyline::isVisible, Polyline::setVisible
  • Object3DEvent::flags

Fixed bug which prevented ShaderCompilationError occurrences from being reported properly. Also fixed GenericVolume::getBuffer.

API Version 2.3

The main changes of this release are the introduction of abstract renderer generalizations and the re-organization of namespaces. Concerning the former, refer to the Overall Design Notes section for details. When upgrading from erlier versions, modify all renderer and visualization instantiations by prefixing the instantiated class with Default, i.e. 

 VolumeVisualization* const view = new VolumeVisualization();

becomes 

 VolumeVisualization* const view = new DefaultVolumeVisualization();

and 

 SlicePlaneRenderer* const view = new SlicePlaneRenderer( module, provider );

becomes 

 SlicePlaneRenderer* const view = new DefaultSlicePlaneRenderer( module, provider );

Further changes:

  • The concept of ''controller decorators'' has been renamed to the more adequate term ''controller filters''.
  • Scene has been renamed to base::view::SceneProvider
  • Model has been renamed to base::model::Scene
  • Model::mask has been renamed to base::model::Scene::volumeMask
  • Model::setMask has been renamed to base::model::Scene::setVolumeMask
  • Model::unsetMask has been renamed to base::model::Scene::removeVolumeMask
  • Model::hasMask has been renamed to base::model::Scene::hasVolumeMask
  • Model::maskExchanged has been renamed to base::model::Scene::volumeMaskExchanged
  • All signals and slots use full-qualified argument types now, so it is not necessary to use typedef- or using-directives when connecting those any longer.

API Version 2.4

The offscreen rendering module has been moved from Carna::base::view to Carna::base:

  • Carna::base::view::OffscreenDisplay now is Carna::base::OffscreenDisplay
  • Carna::base::view::OffscreenVisualizationEnvironment now is Carna::base::OffscreenVisualizationEnvironment

New entities:

  • Methods for voxel-value to HU-value conversion and vice-versa have been added to GenericVolume:
    • base::model::GenericVolume::bufferValueToHUV
    • base::model::GenericVolume::HUVToBufferValue
  • An API extension considering the computation of histograms has been added:
    • base::model::Counter
    • base::model::IndexedCounter
    • base::model::AssociativeCounter
    • base::model::Histogram
    • base::model::VolumeHistogramVolumeAdapter
    • base::model::VolumeHistogram
    • base::model::HistogramClassDetector

Bug fixes:

  • Fixed bug within the VolumeRenderings::AbsorptionProjection::AbsorptionProjection::createControllerUI "controller UI of the AbsorptionProjection" considering a missing signal.
  • The result of the GLEW initialization routine is now checked before performing any other GL operations.

Further changes:

  • Carna::base::glTranslate, Carna::base::glMultMatrix and Carna::base::glLoadMatrix have been moved to the global namespace since they are overloads of functions from the standard GL.h header.
  • Moved Carna::base::Vector3 operators to global namespace.
  • Dropped Carna::base::clearLayout for it wasn't used any more.
  • Splitted common.h into Math.h, Memory.h and Text.h.
  • The "absorption projection" sub-module of the VolumeRenderings module has been renamed to direct volume rendering, for this is the term mainly used for it in the literature.
  • Renamed Object3DNullFilter to Object3DUniformFilter, cause this fits better what the class does.
  • Renamed SceneProvider::ResourceContext to SceneProvider::ResourceClient.
  • Renamed ExpendableGroupBox to ExpandableGroupBox for the class name contained a typo.
  • Moved base::RotatingColor to base::qt::RotatingColor.
  • Moved Scene::max3DTextureSize to SceneProvider::max3DTextureSize.

API Version 2.5

Added:

  • base::model::SceneFactory::setMaximumAllowedSideLength
  • base::model::SceneFactory::maximumAllowedSideLength
  • VolumeRenderings::DVR::DirectVolumeRendering::commitColorMap
  • base::qt::Display::setModule
  • base::qt::Display::hasModule

Fixes:

  • The image returned by OffscreenDisplay::framebuffer is flipped properly now.
  • The DefaultCamera class behaves properly now when used with scenes of different physical dimensions.

Other changes:

  • Dropped the model argument of the AxialPlaneController constructor.
  • The FlowLayout.h is a public header file now.
  • The macro CARNA_RELEASE_VERSION has been renamed to CARNA_PATCH_VERSION.
