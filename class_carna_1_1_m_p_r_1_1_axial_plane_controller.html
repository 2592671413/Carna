Detailed Description

AxialPlaneVisualization sufficient specialization of base::controller::DefaultInteractionStrategy

This implementation disables the user's ability to rotate the camera. As a consequence, the perspective must be set explicitly.

Denoting clicked coordinates

The AxialPlaneController class provides a pickPosition method. This method transforms widget-space coordinates into volume or model space. The code below shows an example of how to extend the existing AxialPlaneController implementation to make it denote the volume-space coordinates the user clicks by spawning points at those coordinates:

 class MyAxialPlaneController : public Carna::MPR::AxialPlaneController
 {
 public:
     MyAxialPlaneController
         ( Carna::MPR::AxialPlaneVisualization& module
         , Carna::base::model::Scene& model
         , const ViewDirection& orientation
         , const Carna::base::Flags< CursorCapabilities >& cursorMode
             = Carna::base::Flags< CursorCapabilities >( adaptAxis ).or( adaptObjects ) );

     virtual void event( Carna::base::Visualization& sourceModule, QEvent* event ) override;
 }

The interface above might be implemented as follows. In the presented code snippet the new functionality of spawning Point3D instances is enabled by holding the SHIFT key. Otherwise, the default functionality is used by delegating the event to Carna::MPR::AxialPlaneController::event.

 MyAxialPlaneController::MyAxialPlaneController
         ( Carna::MPR::AxialPlaneVisualization& module
         , const ViewDirection& orientation
         , const Carna::base::Flags< CursorCapabilities >& cursorMode )
     : Carna::MPR::AxialPlaneController( module, orientation, cursorMode )
 {
 }

 void MyAxialPlaneController::event( Carna::base::Visualization& sourceModule, QEvent* event )
 {
     switch( event->type() )
     {
         case QEvent::MouseButtonPress:
         {
             QMouseEvent* const mouseEvent = static_cast< QMouseEvent* >( event );
             if( mouseEvent->modifiers() & Qt::ShiftModifier )
             {
                 const auto pickedPosition = pickPosition( mouseEvent->x(), mouseEvent->y() );
                 new Carna::base::view::Point3D( model, pickedPosition.toVolumeSpace() );
                 event->accept();
                 return;
             }
             break;
         }
     }
     Carna::MPR::AxialPlaneController::event( sourceModule, event );
 }

When we have our extended AxialPlaneController implementation done, it is time to make an AxialPlaneVisualization actually use it in place of the default controller:

 static void adjustInteractionStrategy
         ( Carna::MPR::AxialPlaneVisualization& visualization
         , const ViewDirection& orientation )
 {
     visualization.doAfterInitialization( [visualization]
         {
             visualization.setController( new MyAxialPlaneController
                 ( visualization
                 , visualization.renderer().provider.scene
                 , orientation ) );
         }
     );
 }
Attention:
Note that the visualization's methods makeFrontView, makeLeftView and makeTopView override the previously set controller. Hence it is convenient to either
  • ensure that the above function adjustInteractionStrategy is invoked after the the perspective was set up,
  • or create an own AxialPlaneVisualization implementation by deriving from DefaultAxialPlaneVisualization and override these methods to invoke adjustInteractionStrategy right after calling the original makeFrontView, makeLeftView or makeTopView implementation.

int  widgetSpaceY

Transforms the given widget-space coordinates into volume or model space.

Example:

 virtual void event( Carna::base::Visualization& sourceModule, QEvent* event ) override
 {
     if( event->type() == QEvent::MouseButtonPress )
     {
         QMouseEvent* const mouseEvent = static_cast< QMouseEvent* >( event );

         const Carna::base::Vector pickedPosition = pickPosition
                 ( mouseEvent->x(), mouseEvent->y() ).toVolumeSpace();

         qDebug()
             << pickedPosition.x() << ", "
             << pickedPosition.y() << ", "
             << pickedPosition.z();
     }

     Carna::MPR::AxialPlaneController::event( sourceModule, event );
 }

The arguments widgetSpaceX and widgetSpaceY must be given in pixels.
