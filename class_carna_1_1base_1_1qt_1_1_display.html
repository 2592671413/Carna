Detailed Description

Simple widget that's only purpose is embedding any given visualization into a Qt based user interface.

All displays do share GL objects - such as textures or shaders - among their contexts.

If you need to grab the rendering from the display, you must ensure that the rendering is finished by the moment you call grabFrameBuffer: 

 QApplication::processEvents();
 display->updateGL();
 QImage framebuffer = display->grabFrameBuffer();

void Carna::base::qt::Display::setModule ( Visualization *  module)

Sets the visualization module that is to be shown within this display.

Precondition:
module != nullptr module->isInitialized() == false

If there was another module set previously, either through this method or the constructor, it is destroyed before the method finishes.

The visualization module will be initialized with the scene provider that was set by the constructor, hence the visualization's renderer will be created with the same. The initialization will be performed immediately if and only if this display already has become visible once, or when the display becomes visible for the first time: this will be the moment that initializeGL is invoked.
