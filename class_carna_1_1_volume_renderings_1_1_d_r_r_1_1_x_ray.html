Detailed Description

VolumeRendererMode that implements Digital Radiograph Reconstruction (DRR).

Usage

An example is provided within the documentation of VolumeRenderer.

Implementation Details

The algorithm is based on ray marching. For each screen fragment, a ray is casted through the volume in order to determine the final pixel intensity \(I\) at this particular screen pixel:

Lets define the intensity

\[ i(x) = \frac{ \mathrm{hu}(x) + 1024 }{ 4096 } \in [0,1] \]

and the attenuation

\[ \mu(x) = \mu_\text{water} \cdot \bigg( 1 + \frac{ \mathrm{hu}(x) }{ 1000 } \bigg) \]

with the constant \(\mu_\text{water}\).

Then we can compute the pixel intensity as

\[ I = I_0 \cdot \exp\bigg( -\int\limits_{x_0}^{x_1} \delta\big(i(x)\big) \cdot \mu(x)\ \mathrm dx \bigg) \]

where

\[ \delta(i) = \begin{cases} 0 & \text{ if }\quad i_\text{lower} > i \\ 1 & \text{ if }\quad i_\text{lower} \leq i < i_\text{upper} \\ k & \text{ if }\quad i_\text{upper} \leq i \end{cases} \]

with \( k \) being some arbitrary positive factor.

Thus \( \mu(x) \) is scaled differently based on which one out of three zones it is located in:
